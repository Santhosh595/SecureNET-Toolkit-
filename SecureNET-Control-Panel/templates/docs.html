FileGuard

AES-256 file encryption with SHA-256 integrity verification

What It Does

Encrypts and decrypts files using AES-256-GCM with authenticated encryption. Supports integrity verification via SHA-256 checksums and secure key derivation.

When to Use

Before storing sensitive files in cloud storage, sharing confidential documents, or creating encrypted backups.

CLI Usage

$ fileguard encrypt ./report.pdf --output ./report.pdf.enc [OK] Encrypted: report.pdf.enc (SHA-256 verified) $ fileguard decrypt ./report.pdf.enc --output ./report.pdf [OK] Decrypted: report.pdf (integrity verified)

Dashboard Walkthrough

Upload files via the web interface, select encryption mode, and monitor progress in real-time. The dashboard shows encryption history and integrity status.

Common Use Cases

  • Encrypting database backups before offsite storage
  • Securing configuration files containing credentials
  • Creating encrypted archives for compliance

Tips

Use the "Verify After Encrypt" option for critical files. Keep your key file separate from encrypted data.

Network Sniffer

Real-time packet capture and intrusion detection system

What It Does

Captures network packets in real-time, analyzes protocols, detects anomalies, and alerts on suspicious activity like port scans, DoS attempts, and data exfiltration.

When to Use

During incident response, network troubleshooting, monitoring for unauthorized access, or investigating suspicious traffic patterns.

CLI Usage

$ netsniff -i eth0 --filter "port 80" -c 1000 [CAPTURE] 1000 packets captured on eth0 [ALERT] Port scan detected from 45.33.32.156

Dashboard Walkthrough

View live packet stream, protocol distribution charts, top talkers, and alert timeline. Filter by protocol, IP, or port.

Common Use Cases

  • Detecting port scanning and reconnaissance activity
  • Identifying unusual data transfer patterns
  • Monitoring for known attack signatures

Tips

Run with elevated privileges for raw packet capture. Use BPF filters to reduce noise on busy networks.

HeaderScan

HTTP security header analyzer with scoring

What It Does

Analyzes HTTP response headers for security best practices, checks for 15+ security headers, and provides a graded score (A-F) with actionable recommendations.

When to Use

During web application security assessments, CI/CD pipeline checks, or when hardening web server configurations.

CLI Usage

$ headerscan https://example.com --format json {"url": "https://example.com", "grade": "B", "score": 78, "missing": ["X-Frame-Options", "Content-Security-Policy"]}

Dashboard Walkthrough

Enter a URL, view the header analysis report with severity indicators, see which headers are present/missing, and get remediation steps.

Tips

Schedule regular scans for production endpoints. Compare scores over time in the Analytics dashboard.

PortMap

Multi-threaded port scanner with service detection

What It Does

Scans TCP/UDP ports across IP ranges using multi-threaded SYN scans, identifies open ports, and detects running services via banner grabbing.

When to Use

During network reconnaissance, asset inventory, vulnerability assessments, or verifying firewall rules.

CLI Usage

$ portmap 10.0.0.1-100 --threads 50 -sV [SCAN] 10.0.0.22:22 (SSH), 80 (HTTP), 443 (HTTPS) [SCAN] 10.0.0.33:3306 (MySQL), 8080 (HTTP-Proxy)

Dashboard Walkthrough

Define target ranges, select scan type (SYN, UDP, Comprehensive), view results in a sortable table with service details.

Tips

Use lower thread counts on production networks. Always get authorization before scanning.

ARPWatch

Real-time ARP spoofing detection

What It Does

Monitors ARP traffic on the local network, maintains a baseline of IP-to-MAC mappings, and alerts immediately when spoofing is detected.

When to Use

On any network where man-in-the-middle attacks are a concern — especially shared networks, guest Wi-Fi, or sensitive environments.

CLI Usage

$ arpwatch --interface eth0 --alert-email admin@company.com [MONITOR] Watching ARP on eth0... [ALERT] ARP Spoof: 192.168.1.1 MAC changed!

Dashboard Walkthrough

View live ARP table, see historical MAC changes, and get instant alerts with severity classification.

Tips

Whitelist known DHCP servers to reduce false positives. Run on each network segment separately.

SubProbe

Subdomain enumerator with takeover detection

What It Does

Enumerates subdomains using DNS brute-force and certificate transparency logs, then checks each for potential subdomain takeover vulnerabilities.

When to Use

During bug bounty hunting, attack surface mapping, or periodic security assessments of your organization's external footprint.

CLI Usage

$ subprobe -d example.com --takeover --threads 20 [FOUND] dev.example.com → CNAME → azurewebsites.net (TAKEOVER RISK) [FOUND] staging.example.com → A → 10.0.0.55 (alive) [FOUND] api.example.com → CNAME → elb.amazonaws.com

Dashboard Walkthrough

Enter a domain, view discovered subdomains in a tree view, see takeover risks highlighted in red, and export results.

Tips

Use custom wordlists for better coverage. Check takeover findings manually before reporting.

JWTInspect

JWT security analyzer with 8 vulnerability tests

What It Does

Decodes JWT tokens, validates signatures, and tests for 8 common vulnerabilities including weak secrets, algorithm confusion, and expired tokens.

When to Use

When reviewing authentication implementations, testing API security, or auditing third-party JWT usage.

CLI Usage

$ jwtinspect eyJhbGciOiJIUzI1NiJ9... [DECODED] alg: HS256, exp: 1700000000 [VULN] Weak secret: "password123" (cracked in 0.3s) [VULN] Token expired: yes (3 days ago)

Dashboard Walkthrough

Paste a JWT token, see decoded header/payload, vulnerability report with severity ratings, and recommendations.

Tips

Always test with production-like tokens (same algorithm/secret strength). Never paste production secrets into dev tools.

TLScan

SSL/TLS security scanner with A-F grading

What It Does

Evaluates SSL/TLS configuration including certificate validity, cipher suites, protocol versions, and known vulnerabilities. Grades from A (best) to F (fail).

When to Use

When deploying new TLS configurations, auditing existing setups, or verifying compliance with security standards.

CLI Usage

$ tlscan example.com --port 443 [GRADE] B — TLS 1.3, Valid cert, Strong ciphers [WARN] TLS 1.0 still enabled (deprecated)

Dashboard Walkthrough

Enter hostname and port, view certificate chain, cipher suite details, and grade breakdown with improvement suggestions.

Tips

Scan both HTTP and HTTPS ports. Re-scan after configuration changes to verify fixes.

LogSentry

Multi-source log analyzer and threat detector

What It Does

Aggregates logs from multiple sources (syslog, auth logs, application logs), detects threat patterns using signature matching, and generates alerts.

When to Use

For centralized log monitoring, detecting brute-force attacks, identifying policy violations, and correlating events across systems.

CLI Usage

$ logsentry --watch /var/log/auth.log --rules brute-force,brute-force [ALERT] 14 failed SSH attempts from 203.0.113.42 in 5 minutes [INFO] Pattern matched: "brute-force" (confidence: 95%)

Dashboard Walkthrough

View log stream in real-time, filter by source/severity, see pattern match alerts, and configure custom detection rules.

Tips

Add custom regex rules for your specific threat models. Use the timeline view to correlate events.

SecretSniff

Hardcoded secret and API key scanner

What It Does

Scans source code, configuration files, and directories for hardcoded secrets, API keys, tokens, and credentials using regex patterns and entropy analysis.

When to Use

Before committing code, during code reviews, auditing third-party dependencies, or rotating credentials after a suspected leak.

CLI Usage

$ secretsniff ./src/ --exclude node_modules --format json [HIGH] config/production.yml:42 — AWS_ACCESS_KEY_ID=AKIA... [MED] .env:7 — DATABASE_URL contains password

Dashboard Walkthrough

Select scan target, configure exclusions, view findings with masked values, and generate remediation reports.

Tips

Run as a pre-commit hook. Add custom patterns for your organization's internal services.

DNSAudit

Comprehensive DNS security auditor

What It Does

Audits DNS configurations including DNSSEC, SPF, DKIM, DMARC records, zone transfer security, and checks for DNS hijacking indicators.

When to Use

During domain security assessments, after DNS changes, or as part of regular security audits.

CLI Usage

$ dnsaudit example.com --check dnssec,spf,dmarc [FAIL] DNSSEC: Not configured [PASS] SPF: v=spf1 include:_spf.google.com ~all [WARN] DMARC: p=none (relaxed policy)

Dashboard Walkthrough

Enter domain, view DNS record analysis, see security recommendations, and compare against best practices.

Tips

Audit all domains regularly. Pay special attention to expired or misconfigured DNSSEC keys.
